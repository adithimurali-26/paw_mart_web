📡 REST vs SOAP

Understanding the two major web service paradigms

🟢 REST (Representational State Transfer)

REST is a lightweight, stateless architecture using standard HTTP methods (GET, POST, PUT, DELETE). Data is typically JSON. Easy to consume with fetch() in JavaScript.

fetch('https://api.pawmart.com/pets') .then(r => r.json()) .then(data => console.log(data))

🟡 SOAP (Simple Object Access Protocol)

SOAP is a protocol using XML envelopes for messaging. It's more rigid and verbose but supports enterprise features like WS-Security and ACID transactions.

<soapenv:Envelope> <soapenv:Body> <getPet> <id>1</id> </getPet> </soapenv:Body> </soapenv:Envelope>

📄 WSDL (Web Services Description Language)

WSDL is an XML document that describes a SOAP web service – like a contract defining methods, parameters, and endpoints. Think of it as the API documentation for SOAP services.

🔐 Authentication Concepts

API Key: A static secret token sent in headers. Simple but less secure.

JWT (JSON Web Token): A signed, Base64-encoded token containing claims (user ID, expiry). Stateless and scalable.

// JWT Header Authorization: Bearer eyJhbGci... // Payload (decoded) { "sub":"user42","exp":1700000000 }
